Phần 2 - Thực thi: Báo cáo Phân tích Kiến trúc Hệ thống
Báo Cáo: Lựa Chọn Web Service Cho Hệ Thống Tài Chính & Chuyển Đổi Số

I. Quyết định kiến trúc & Lý do cụ thể

Sử dụng SOAP cho Legacy Core Banking: Hệ thống core banking quản lý tiền và các giao dịch tài chính nhạy cảm. SOAP cung cấp WS-AtomicTransaction giúp đảm bảo các giao dịch phân tán (distributed transactions) tuân thủ nguyên tắc ACID (Atomicity, Consistency, Isolation, Durability) trên nhiều cơ sở dữ liệu. Cấu trúc WSDL khắt khe giúp đảm bảo tính toàn vẹn dữ liệu, không cho phép client gửi các yêu cầu sai định dạng.

Sử dụng REST cho Hệ thống Microservices mới: Nhóm ứng dụng này phục vụ người dùng cuối (B2C), yêu cầu tốc độ phản hồi tính bằng mili-giây và phải xử lý hàng nghìn kết nối đồng thời. Việc sử dụng REST kết hợp định dạng JSON giúp giảm thiểu băng thông, dễ dàng mở rộng ngang (horizontal scaling) do tính chất phi trạng thái (stateless), và giúp các team Frontend/Mobile phát triển tính năng cực nhanh.

II. Khuyến nghị cho công ty

Duy trì song song và Phân tách vùng (Segregation): Không cố gắng ép hệ thống Core Banking đập đi xây lại bằng REST, cũng không bắt các ứng dụng Mobile gọi trực tiếp vào hệ thống SOAP. Hãy để hai hệ thống làm tốt nhất công việc của chúng.

Triển khai API Gateway (Bức tường trung gian): Khuyến nghị công ty xây dựng một API Gateway ở giữa. Mobile App sẽ gọi REST API gọn nhẹ đến API Gateway. Sau đó, API Gateway sẽ đóng vai trò chuyển đổi (translate) từ REST/JSON sang SOAP/XML để giao tiếp an toàn với hệ thống Legacy ở phía sau.

Chuyển đổi dần dần (Strangler Pattern): Các module phụ trợ ít quan trọng trong Core Banking có thể được bóc tách dần và viết lại thành Microservices dùng REST, giảm tải từ từ cho hệ thống Legacy.

III. Thuật ngữ & Khái niệm Cốt lõi (Glossary)

SOAP (Simple Object Access Protocol): Là một giao thức (protocol) truyền tải thông điệp chính thức.

XML (eXtensible Markup Language): Định dạng dữ liệu duy nhất mà SOAP sử dụng, có tính mô tả cao nhưng nặng.

WSDL (Web Services Description Language): Hợp đồng bằng XML mô tả chi tiết Web Service cung cấp những hàm gì, đầu vào/đầu ra ra sao. Client dựa vào đây để gọi hàm.

WS-Security: Gói tiêu chuẩn bảo mật đính kèm trong chuẩn SOAP, giúp mã hóa và ký điện tử trực tiếp trên từng đoạn dữ liệu XML.

REST (Representational State Transfer): Là một kiểu kiến trúc (architectural style), không phải giao thức. Nó đưa ra các nguyên tắc thiết kế ứng dụng mạng.

Resource (Tài nguyên): Trái tim của REST. Mọi dữ liệu (khách hàng, tài khoản) được xem là một tài nguyên và được định danh bằng một URI (Uniform Resource Identifier) duy nhất.

HTTP Methods: REST tận dụng các hành động tiêu chuẩn của HTTP bao gồm GET (Lấy), POST (Tạo), PUT (Cập nhật), DELETE (Xóa) để thao tác với Resource.

Stateless (Phi trạng thái): Nguyên tắc cốt lõi của REST. Server không lưu trữ bất kỳ thông tin nào về client giữa các request. Mỗi request phải chứa đầy đủ thông tin để server hiểu và xử lý, giúp hệ thống dễ dàng scale.

JSON (JavaScript Object Notation): Định dạng dữ liệu phổ biến nhất dùng trong REST, nhẹ hơn XML và trình duyệt/mobile có thể phân tích cú pháp cực kỳ nhanh.
